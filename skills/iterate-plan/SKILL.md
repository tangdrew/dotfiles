# Iterate on Plan

Revise an existing implementation plan based on new information or feedback.

## Process

### 1. Read the Current Plan
- Read the existing plan document thoroughly
- Understand what has and hasn't been implemented yet
- Note the original reasoning behind decisions

### 2. Gather New Context
- Ask the user what changed or what feedback they received
- Research any new information in the codebase
- Identify which parts of the plan are affected

### 3. Propose Revisions
- Clearly state what you're changing and why
- If the change affects downstream phases, trace through the impacts
- Present the updated sections for review

### 4. Update the Plan
- Edit the plan document with the revisions
- Mark any completed phases
- Add a revision note at the top with the date and summary of changes

## Rules
- **Preserve completed work** — don't rewrite phases that are already implemented unless specifically asked
- **Trace downstream impacts** — a change in Phase 1 may invalidate Phase 3
- **Explain your reasoning** for each revision
- Keep the same document structure; don't reorganize unnecessarily
